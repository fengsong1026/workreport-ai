# 【学习工作赛道】一个应用解决打工人汇报问题——WorkReport AI：全岗位智能工作汇报平台

**标签：** #学习工作

---

## 1. 创意名称 + 创意介绍

**创意名称：WorkReport AI——让汇报不再是负担**

**想解决什么问题：**
无论是程序员、产品经理、销售还是运营，每周最让人头疼的不是工作本身，而是"把干的事说给领导听"——写汇报这件事费时、费脑、费情绪，却几乎产生不了任何实际价值。

**为什么会想到做这个：**
作为开发者，亲历了每周翻 Git 记录写周报的痛苦之后，萌生了一个想法：每个岗位的工作都有"数字痕迹"——代码提交、任务单、日历记录、文档编辑……这些数据天然就是汇报的原料，只缺一个能自动"翻译"成汇报语言的 AI 管道。

**大概是什么产品：**
一个面向职场全岗位的 AI 工作汇报自动化平台，通过接入不同的工作数据源（Git / 任务系统 / 日历 / 文档等），自动聚合当周工作轨迹，由 AI 生成结构化的中文日报/周报/月报。

---

## 2. 目标用户及痛点

**面向哪些用户：**
所有需要定期向上汇报的职场人——程序员、产品经理、设计师、销售、运营，以及需要管理团队汇报的 Team Lead 和中层管理者。

**在什么场景下使用：**
- 每周五/月底，需要整理汇报材料时
- 多线并行，工作分散在多个工具中，难以汇总
- 团队管理者需要批量收集成员周报、生成团队进展摘要

**当前痛点：**
没有这个产品，用户要手动回忆一周干了什么、翻多个工具的历史记录、用自然语言重新描述、再套公司模板格式——平均耗时 30–60 分钟，质量全靠个人表达能力，漏记、写得干瘪、不够"好看"是常态。

---

## 3. 子应用架构（已落地 Demo）

平台采用"数据源插件化"架构，每类岗位对应一个数据采集子应用：

| 子应用 | 数据源 | 目标用户 | 状态 |
|--------|--------|----------|------|
| **Git Weekly Automation** | Git 提交记录 | 程序员 / 技术研究员 | ✅ 已完成 |
| Task Reporter | Jira / Linear / 飞书任务 | 产品 / 项目管理 | 规划中 |
| Calendar Reporter | Google Calendar / 企业微信 | 所有岗位 | 规划中 |
| Doc Reporter | Notion / 语雀 / Confluence | 内容 / 运营 | 规划中 |

### 已落地子应用：Git Weekly Automation

通过全局 Git Hook 实时捕获所有仓库的提交记录，结合 AI（OpenAI / DeepSeek / Ollama 等兼容 API）自动生成格式规范的中文工作周报，支持 macOS 定时调度，真正做到"提交即汇报，周五自动交卷"。

**核心亮点：**
- **零侵入采集**：全局 Git Hook，安装后对现有工作流透明
- **多仓库支持**：自动扫描目录树，含 submodule，commit hash 去重
- **AI 生成**：对接 OpenAI / DeepSeek / Ollama 等任意兼容 API，输出简体中文
- **定时调度**：macOS launchd 托管，每周五 17:00 自动触发，无需手动操作
- **模板可定制**：`{{占位符}}` 语法，按团队/公司汇报格式自由配置

---

## 4. 价值与意义

**效率提升：**
以 Git Weekly Automation 子应用为例，安装后对日常工作流完全透明——每次 `git commit` 自动采集，周五定时触发，将原本 30–60 分钟的周报撰写压缩为 0 人工干预。平台级别扩展后，非技术岗位同样能获得同等体验。

**商业价值：**
周报管理是企业管理成本的隐性黑洞，尤其在中大型研发和运营团队中。WorkReport AI 可延伸为 SaaS 团队效能产品：自动汇聚全员工作数据，按项目/人员/部门维度生成多层次汇报，帮助管理者低成本掌握团队动态，减少无效同步会议。
